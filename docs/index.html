Version 1 · Local Reference Core

A complete local comparison workflow.

Version 1 establishes the real-time audio core and the extensible architecture behind ReferenceLab. It runs as a stereo VST3 or standalone application on Windows 10/11.

Implementation complete. Formal multi-DAW, performance and calibrated-meter validation remains in progress before a public release.

01

Reference library

WAV, AIFF, FLAC and MP3 files, recursive scanning, editable Unicode metadata, search, filters, sorting, missing-file recovery and an atomic versioned JSON library.

02

Click-safe A/B playback

Play, pause, stop, seek, start offsets and crossfaded A/B loops, with sample-rate adaptation and rapid Mix/Reference switching.

03

Fair level matching

K-weighted gated loudness analysis, automatic or manual gain compensation, smoothing and a safety limit help prevent “louder sounds better” bias.

04

Focused listening

Symmetric high-pass, bell and low-pass comparison EQ, plus Stereo, Mono, Mid and Side monitoring applied consistently to both sources.

05

Technical analysis

FFT overlay, oscilloscope, LUFS, Peak, True Peak, RMS, Crest, LRA, correlation and stereo-width views provide context for listening decisions.

06

Built for DAW safety

Asynchronous loading, configurable LRU cache, project-state restoration and Safe Export keep reference audio out of detected offline renders.

Version 2 · Product vision

From a local library to an extensible reference ecosystem.

V2 builds on the existing multi-reference model and provider abstraction. The audio core remains stable while new sources and a more flexible interface are added around it.

  1. 01

    Extended sources

    Importable JSON catalogues, direct HTTP/HTTPS audio sources and isolated provider errors.

  2. 02

    Multiple libraries & playlists

    Organise different collections, build playlists and move quickly between several active references.

  3. 03

    Full waveform workflow

    Overview waveform, draggable start/current/end markers, highlighted loop selection, zoom, scroll and stored positions.

  4. 04

    Advanced DAW sync

    More precise transport following, offsets and robust fallback when host timing information is unavailable.

  5. 05

    Professional adaptive UI

    Resizable and collapsible panels, consistent Mix/Reference colour language, readable FFT/EQ overlays and saved layouts.

Proposed V2 exploration

Cambridge MT catalogue access

ReferenceLab is interested in discussing a respectful, technically sustainable way to help users discover and access suitable Cambridge MT resources from within the V2 reference workflow.

  • Integration only with explicit technical and legal approval.
  • Clear source attribution and compliance with applicable licences and terms.
  • Metadata and public links first; no rehosting or redistribution without permission.
  • User-initiated access, resilient provider isolation and straightforward removal if required.

Cambridge MT is not currently integrated with, affiliated with, or endorsing ReferenceLab. This section describes a proposed discussion for Version 2.

Product principles

01

Listen first

Measurements support critical listening; they do not replace it.

02

Compare fairly

Matched loudness and identical monitoring processing reduce bias.

03

Protect the session

Real-time safety, graceful errors and Safe Export are core behaviours.

04

Respect the source

Online catalogues require permission, attribution and licence compliance.
